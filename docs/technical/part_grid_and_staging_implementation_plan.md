# 技术设计方案：装配页面零件格子与暂存区实现计划 (确认版)

## 1. 核心概念与区域划分 (Zone Definition)

根据 `product_documentation.md` 与 `workbench.ts` 的定义，装配页面将划分为三个物理与逻辑隔离的区域，采用**点击式选点**作为核心交互。

| 区域名称 | ZoneType | 交互职责 | 物理特性 |
| :--- | :--- | :--- | :--- |
| **主竞技场 (Active Arena)** | `ACTIVE_ARENA` | 核心组装区。第一个落位零件自动成为 ROOT。 | 参与物理仿真，导出 URDF。 |
| **暂存工作台 (Workbench)** | `WORKBENCH` | 存放拆解零件。支持在槽位中进行 **360° 旋转预览**。 | 禁用全局物理，网格对齐。 |
| **物料库 (Part Grid)** | `PREVIEW` | 侧边栏列表。点击零件后弹出 **360° 旋转预览窗口**。 | 纯 UI 渲染，无世界坐标。 |

## 2. 功能组件设计

### 2.1 零件格子 (Part Grid / Material Library)
*   **数据模型**：
    - 调用后端 `/api/get_verified_parts` 获取已验证且包含端口定义的 Technic 零件列表（包含 ID 和 2D 预览图）。
*   **交互逻辑 (Pick & Snap Flow)**：
    1.  **零件选择**：用户在侧边栏点击零件图标。
    2.  **强制预览**：弹出焦点预览窗口（支持鼠标旋转缩放），显示零件所有端口。
    3.  **源端口锁定**：用户在预览窗口点击某个端口（源端口）。
    4.  **落位拼装**：
        - 若场景不为空：用户点击主竞技场内已有零件的端口（目标端口），零件自动滑入拼装。
        - 若场景为空：点击主竞技场空白处，该零件直接在原点落位并成为场景 ROOT（地基）。
*   **设计原则**：**禁绝拖拽**。所有拼装动作必须经过“源端口选择 -> 目标端口选择”的显式确认。

### 2.2 暂存托盘 (Staging Tray)
- **UI 实现**: 在右侧侧边栏引入 `StagingTrayPanel.tsx`，采用垂直列表展示。
- **逻辑实现**: 使用 `StagingGrid` 管理 $5 \times 5$ 的逻辑槽位。
- **交互流程**: 
    1. 双击场景零件 -> 调用 `stagePart(id)`。
    2. 零件从 `ACTIVE_ARENA` 移除连接，其 `zone` 变为 `STAGED`。
    3. 零件通过世界坐标映射到逻辑槽位，并在右侧侧边栏列表中实时出现。
点击主竞技场零件端口 -> 执行从暂存区到竞技场的位移补间动画。
*   **拆解保护**：从主组装体上双击拆下的独立部分（Island），若无其他连接，将自动执行“回弹”动画滑行进入暂存区的空余槽位。

## 3. 实现路线图 (Roadmap)

### 第一阶段：后端 API 支持
- [ ] **接口开发**：实现 `GET /api/get_verified_parts`，返回所有状态为 `verified` 的零件摘要（ID, 预览图 URL, 端口数）。

### 第二阶段：前端 360° 交互式预览 (Interactive Focus)
- [ ] **预览图层 (Focus Overlay)**：在 UI 层创建独立的 Canvas/Viewport，用于渲染选定零件的 360° 预览。
- [ ] **端口选点控制器**：确保预览窗内的点击能正确识别 `portId` 并将其锁定为 `sourceLockData`。

### 第三阶段：拼装流与空场景逻辑 (Logic Flow)
- [ ] **FSM 状态链补齐**：`IDLE -> SELECTING_SOURCE_PORT -> SELECTING_TARGET_PORT -> ANIMATING`。
- [ ] **地基初始化逻辑**：当 `store.parts` 为空时，拦截第一个 Snap 动作，直接将 Source 移至场景中心并标记为 `ZoneType.ACTIVE_ARENA`。

### 第四阶段：暂存区视觉与动画 (Visuals)
- [ ] **Workbench 辅助栅格**：在 `Scene.jsx` 中添加暂存区的底板渲染。
- [ ] **回弹补间动画**：实现拆解零件向暂存区滑行的物理反馈动画。

---

## 4. 负面约束与边界说明 (Negative Constraints)

1.  **严禁拖拽入场**：零件从物料库进入场景的唯一方式是通过端口匹配（Snap）或空场景落位点击。
2.  **单一焦点**：同时只能有一个零件处于“源端口待选”或“目标点寻找”状态。
3.  **槽位锁定**：暂存区零件仅支持在槽位中心旋转，不支持槽位间的自由平移，以维持画面整洁度。
