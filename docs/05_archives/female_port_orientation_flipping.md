# Issue: 梁/孔（FEMALE）端口朝向反转分析报告

## 问题背景
在修复了插销（MALE）端口的镜像矩阵问题后，插销的紫色箭头已能正确指向零件外部。但观测发现，梁（Beam）以及所有具有孔位（FEMALE）零件的蓝色箭头指向存在系统性反转：箭头全部指向零件内部，而非开口向外。

## 现状描述
- **现象**：在红色的梁（如 32524.dat）上，每个孔位的蓝色箭头都指向梁的中心。
- **视觉证据**：
    - 上表面的孔：箭头向下（指向梁中心）。
    - 下表面的孔：箭头向上（指向梁中心）。
- **逻辑矛盾**：
    - 根据装配协议，`MALE` 突出方向应与 `FEMALE` 开口方向对冲（Vector Opposite）。
    - 如果 `FEMALE` 箭头向内，则插销必须穿过零件身体才能实现“对冲”，这会导致装配位姿计算错误（零件重叠）。

## 根本原因分析
### 1. LDraw 原件定义偏差
LDraw 的孔位原件（如 `peghole.dat`）其局部坐标系的 `+Y` 轴是沿着孔洞**向内**延伸的。

### 2. `port.py` 中的映射错误
目前的 `port.py` 代码中（约 L34-35 行）定义如下：
```python
# FEMALE（孔）：+Y = 孔接收/开口方向  → 令 Z = +Y
#   normalizer = Rx(-90°)
```
这里错误地假设了 `+Y` 是“开口方向”。实际上，`+Y` 是“深入方向”。为了让 Z 轴（箭头）指向开口外部，应该令 `Z = -Y`。

## 修复建议
需要修改 `port.py` 中的归一化常量：

1.  **修改定义图示**：
    ```python
    # 修改前：FEMALE (Rx-90) -> Z = +Y (向内)
    # 修改后：FEMALE (Rx+90) -> Z = -Y (向外)
    ```
2.  **代码变更**：
    将 `_NORMALIZER_MAP` 中所有 `FEMALE` 类型对应的归一化矩阵从 `_Rx_NEG90` 更改为 `_Rx_POS90`（或者定义一个新的 `_Rx_POS90` 给孔使用，确保其 Z 轴指向 `-Y_ldraw`）。

## 影响评估
- **前端渲染**：修复后，所有蓝色箭头将从零件表面弹射向外，符合物理直觉。
- **自动吸附**：这是实现正确自动吸附（Snapping）的先决条件。只有当两个端口 Z 轴都向外时，计算出的相对变换矩阵才能保证零件不发生重叠。
- **兼容性**：此修改具有通用性，适用于所有基于 `peghole.dat`, `axlehole.dat` 定义的零件。

## 验证计划
1. 修改 `port.py` 中的归一化映射。
2. 重启后端服务。
3. 验证 API 返回的孔位 Z 轴是否相对于零件表面法线向外。
4. 在前端观测蓝色箭头指向。
