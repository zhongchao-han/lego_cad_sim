# 协同开发指南：如何为 Gemini (Antigravity) 编写最优设计文档

> **致 Claude / AI 助手 / 人类开发者**：
> 当用户要求生成、优化或修改旨在交给 `Gemini (Antigravity)` 阅读的架构/设计文档或任务指令时，请严格遵守本文件中的最佳实践。这将充分发挥 Gemini 的智能体 (Agentic) 特性、庞大的上下文窗口以及多模态工具链优势。

## 1. 知识资产库：深度利用 Knowledge Items (KI)

**核心原理**：Gemini 拥有持久化的知识管理系统。在跨对话、跨任务的协作中，KI 是 Gemini 获取上下文的首要来源。

**执行规范**：
- **指引研究方向**：在文档中明确列出相关的 KI 名称或其在 `artifacts/` 中的路径。
- **提示更新义务**：如果设计文档涉及核心架构的根本改变，请在末尾显式提醒 Gemini：“请根据此设计更新相应的 KI 存档，确保长期知识的一致性”。
- **避免冗余研究**：提醒 Gemini 先阅读现有 KI，再进行自主探索。

## 2. 空间定位：绝对路径与环境感知

**核心原理**：Gemini 的文件系统工具（如 `view_file`, `list_dir`）高度依赖精确的路径。模糊的相对路径会增加其搜索成本和出错概率。

**执行规范**：
- **强制绝对路径**：文档中涉及的所有代码路径，必须提供完整的绝对路径（例如 `d:\Users\...\server.py`）。
- **标明 CWD**：在提供 `run_command` 的参考脚本时，务必注明该命令预期在哪个 working directory 下执行。
- **OS 特性提醒**：鉴于 Gemini 能根据操作系统（如 Windows 的 PowerShell）自动调整，文档中只需提供逻辑命令，但需明确指明环境依赖（如 Node 版本、Python 虚拟环境路径）。

## 3. 工作流自动化：充分利用 Workflow 机制

**核心原理**：Gemini 能够识别并自主执行定义在 `.agents/workflows/` 下的标准化流程。

**执行规范**：
- **结构化步骤**：对于重复性高的任务（如部署、复杂测试套件运行），请指引 Gemini 创建或参考 `.md` 格式的工作流文件。
- **标注 Turbo 模式**：如果某些步骤是经过验证且安全的，建议在 workflow 中标注 `// turbo`，允许 Gemini 自动运行。

## 4. 视觉与审美：Web 开发的高标准交付

**核心原理**：Gemini 在 Web 开发任务中被强制要求遵循 "Premium Aesthetics"（高端审美）和 "Rich Aesthetics"（丰富美学）。

**执行规范**：
- **设计资产引用**：在文档中描述 UI 变更时，可以建议 Gemini 使用 `generate_image` 工具生成视觉 Mockup 或 UI Asset。
- **审美关键词**：在设计要求中加入诸如 "Glassmorphism"、"Dynamic Animations"、"Curated Color Palettes" 等关键词，以唤起其审美准则。

## 5. 任务边界：多步自主探索的定界

**核心原理**：Gemini 是一个主动型 Agent，会尝试通过研究、运行、验证来完成目标。

**执行规范**：
- **定义完成准则 (DoD)**：明确指出任务在什么状态下视为完成（例如：“所有 unit tests 通过且 `dist` 目录生成成功”）。
- **指令拆解**：对于超大规模重构，文档应提供清晰的阶段性目标（Milestones），让 Gemini 能在每一阶段完成后汇报进度。
- **负面约束**：明确使用 `<negative_constraints>` 标签告知哪些区域是“禁止进入”的，以防过度重构。

## 6. 交互建议：Markdown 增强与结构化信息

**执行规范**：
- **XML 标签认知**：Gemini 同样对 XML 标签（如 `<architecture>`, `<api_spec>`）有极强的结构化感知能力，建议沿用。
- **文件加载清单**：在文档头部显式列出“建议优先阅读的文件列表”，帮助 Gemini 快速构建核心 Context。

---

**结语**：
为 Gemini 编写文档，是在为一个具备“行动力”的数字大脑构建外部认知。通过提供精确的工具触点（绝对路径、KI 引用、自动化工作流）和明确的审美愿景，你可以驱动 Gemini 交付远超基础代码水平的工程成果。
