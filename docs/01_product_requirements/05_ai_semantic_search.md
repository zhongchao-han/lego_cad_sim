# AI 语义化零件检索引擎 (AI Semantic Part Search)

## 1. 业务愿景 (Business Vision)
随着 LDraw 零件库的不断膨胀，传统的“按准确英文代号或数字 ID 检索”的方式已经对非核心硬核玩家甚至一般设计师构成了巨大的门槛。用户往往只知道“有个多孔的梁”而不知道其官方代号为 `Technic Beam ... Baseplate`。
本项目依托于本地化高性能检索引擎 **MeiliSearch** 与大语言模型（如 DeepSeek）的推理能力，打造了一套**“允许模糊表达、理解受众心智、速度极快”**的全局检索交互面板。

## 2. 核心功能场景 (Core Features)

### 2.1 全局唤醒与沉浸式交互
*   **交互规范**：系统预留了全局快捷键 `Cmd+K` / `Ctrl+K`，可在任何视图（装配视图、复核视图）一键唤醒搜索模态悬浮窗。
*   **极速预览**：摒弃了拖沓的表格选单。搜索命中的零件可以直接在左侧或视觉反馈区被预览，并在组装页面时由底层自动化加入“暂存区”(Staging Tray) 和中央 3D 渲染画布中。

### 2.2 LLM 意图拦截与语义重写
*   **智能降级机制**：当判定用户的词元长度或包含纯中文时（即认为存在语义鸿沟），前端将冻结对底层倒排索引库的直接查表，转由 LLM 代理进行特征抽象（Feature Extraction），并提取 1-4 个标准的 LDraw 规范名词（如 Liftarm, Plate, Technic）。
*   **私有化配置面板**：UI 内置配置选项，给予用户对底层知识引擎架构的绝对控制权：用户可自定义开启/关闭大模型、填充大模型 API Key（支持任何 OpenAI 兼容规范接口）以及设定具体模型版本。密钥全程存储于浏览器 `localStorage`，不触网留后端，保障最高级安全。

### 2.3 MeiliSearch 高性能检索池与同义词联想
*   **毫秒级命中**：底层数据池通过增量同步脚本时刻与本地 LDraw 库保持映射。
*   **乐高黑话字典 (Synonyms)**：引擎在建库期间注入了大量的乐高行业俗名同义词拓展表（如将 `Plate`, `Panel`, `Baseplate`, `Board` 强绑定）。使得大模型即使仅仅输出 `Plate`，底层数据库也知道自动容错并找到 `Baseplate` 归属下的零件。

## 3. SLA 质量底线与异常机制 (SLA & Constraints)
*   **网络断网/超时限宽**：在断网或 LLM 提供商宕机时，UI 不应崩溃，而应迅速降级出具体的错误描述文字（"大模型翻译失败" 等）。
*   **防抖**：所有敲击触发须严格执行设定好的竞态（Race-Condition）防抖（当前为 300-500ms），绝不发多余网络请求。
